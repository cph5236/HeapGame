# iOS Build via CI-Generated Capacitor + TestFlight — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Heap to TestFlight via a macOS GitHub Actions job that generates the `ios/` Capacitor project on a version bump, patches it for portrait/anonymous/ad-free, archives it, and uploads to TestFlight — with no iOS-native code committed to the repo.

**Architecture:** `ios/` is never committed (already gitignored). A macOS runner runs `cap add ios` fresh each build, generates icons, applies an Info.plist patch script (portrait lock, display name, version, monotonic+rerun-safe build number, AdMob test ID to avoid the SDK launch-crash, encryption flag), then fastlane signs via an App Store Connect API key and uploads to TestFlight. A version-diff guard skips the (free, public-repo) macOS job unless `package.json` version changed.

**Tech Stack:** Capacitor 8.2, `@capacitor/ios`, `@capacitor/assets`, GitHub Actions (macOS runner), fastlane, PlistBuddy, Vitest.

**Spec:** [docs/superpowers/specs/2026-06-09-ios-build-design.md](../specs/2026-06-09-ios-build-design.md)

---

## Deviations from spec (conscious, during planning)

- **No `build:ios` npm script.** It would duplicate `build` (web bundle is identical; the cap steps live in CI because `ios/` is ephemeral). The workflow calls `npm run build` directly. DRY.
- **No `capacitor.config.ts` edit.** The current config (`appId`, `appName`, `webDir: dist`) already suffices for iOS; a custom `ios.scheme` would change the WKWebView origin (localStorage keying) for no benefit. Left unchanged.
- **Added `@capacitor/ios` dependency** — required for `cap add ios` to run non-interactively in CI; not in the spec's component list.
- **Added `APPLE_TEAM_ID` secret** — automatic signing needs the Development Team ID; spec listed only the three ASC secrets.

## Task 3: Version-diff guard script (TDD)

**Files:**
- Create: `scripts/ios-version-guard.sh`
- Test: `scripts/__tests__/ios-version-guard.test.ts`

The guard exits the macOS job early unless `package.json` `version` changed versus the previous commit. Logic lives in bash (matches CI idiom in `mobile.yml`); the Vitest test drives it in a throwaway git repo so it runs in `npm test` on Linux. The script writes `should_build=true|false` to `$GITHUB_OUTPUT`, falling back to stdout when that var is unset (so the test can capture it); diagnostics go to stderr.

- [ ] **Step 1: Write the failing test**

Create `scripts/__tests__/ios-version-guard.test.ts`:

```ts
import { describe, it, expect, beforeEach, afterEach } from 'vitest';
import { execFileSync } from 'node:child_process';
import { mkdtempSync, writeFileSync, rmSync } from 'node:fs';
import { tmpdir } from 'node:os';
import { join } from 'node:path';

const SCRIPT = join(__dirname, '..', 'ios-version-guard.sh');

function git(cwd: string, ...args: string[]): void {
  execFileSync('git', args, { cwd, stdio: 'pipe' });
}
function writePkg(cwd: string, version: string): void {
  writeFileSync(join(cwd, 'package.json'), JSON.stringify({ name: 'heap-game', version }) + '\n');
}
function runGuard(cwd: string): string {
  return execFileSync('bash', [SCRIPT], { cwd, encoding: 'utf8' }).trim();
}

describe('ios-version-guard', () => {
  let dir: string;
  beforeEach(() => {
    dir = mkdtempSync(join(tmpdir(), 'ios-guard-'));
    git(dir, 'init', '-q');
    git(dir, 'config', 'user.email', 't@t.t');
    git(dir, 'config', 'user.name', 'tester');
  });
  afterEach(() => rmSync(dir, { recursive: true, force: true }));

  it('builds when the version changed between commits', () => {
    writePkg(dir, '0.2.5'); git(dir, 'add', '.'); git(dir, 'commit', '-qm', 'v1');
    writePkg(dir, '0.2.6'); git(dir, 'add', '.'); git(dir, 'commit', '-qm', 'v2');
    expect(runGuard(dir)).toBe('should_build=true');
  });

  it('skips when the version is unchanged between commits', () => {
    writePkg(dir, '0.2.5'); git(dir, 'add', '.'); git(dir, 'commit', '-qm', 'v1');
    writeFileSync(join(dir, 'other.txt'), 'x'); git(dir, 'add', '.'); git(dir, 'commit', '-qm', 'noop');
    expect(runGuard(dir)).toBe('should_build=false');
  });

  it('builds on the first commit (no previous package.json)', () => {
    writePkg(dir, '0.2.5'); git(dir, 'add', '.'); git(dir, 'commit', '-qm', 'v1');
    expect(runGuard(dir)).toBe('should_build=true');
  });
});
```

- [ ] **Step 3: Write the script**

Create `scripts/ios-version-guard.sh`:

```bash
#!/usr/bin/env bash
# Decide whether the iOS build should run: only when package.json "version"
# differs from the previous commit. Emits `should_build=true|false` to
# $GITHUB_OUTPUT (or stdout when unset). Diagnostics go to stderr.
set -euo pipefail

out="${GITHUB_OUTPUT:-/dev/stdout}"

current="$(node -p "require('./package.json').version")"

if prev_json="$(git show HEAD~1:package.json 2>/dev/null)"; then
  previous="$(printf '%s' "$prev_json" \
    | node -e "let s='';process.stdin.on('data',d=>s+=d).on('end',()=>console.log(JSON.parse(s).version))")"
else
  previous=""
fi

if [ "$current" = "$previous" ]; then
  echo "iOS build: version unchanged ($current) — skipping." >&2
  echo "should_build=false" >> "$out"
else
  echo "iOS build: version ${previous:-<none>} -> $current — building." >&2
  echo "should_build=true" >> "$out"
fi
```

## Task 4: Info.plist patch script

**Files:**
- Create: `scripts/ios-patch-plist.sh`

Applies every native config the regenerated project needs, via `PlistBuddy` (macOS-only — runs in CI, not locally). The local check is a bash syntax parse (`bash -n`); functional verification happens in Task 7's CI run. Build number is `run_number * 100 + run_attempt` so it is monotonic **and** distinct on reruns (Actions sets `GITHUB_RUN_NUMBER`/`GITHUB_RUN_ATTEMPT` automatically).

- [ ] **Step 1: Write the script**

Create `scripts/ios-patch-plist.sh`:

```bash
#!/usr/bin/env bash
# Patch the regenerated Capacitor iOS Info.plist. Runs on the macOS CI runner
# after `cap add ios` + `cap sync ios`. macOS-only (PlistBuddy).
set -euo pipefail

PLIST="ios/App/App/Info.plist"
PB="/usr/libexec/PlistBuddy"

VERSION="$(node -p "require('./package.json').version")"
# Monotonic AND rerun-safe: run_number repeats on a rerun; run_attempt increments.
BUILD="$(( ${GITHUB_RUN_NUMBER:-0} * 100 + ${GITHUB_RUN_ATTEMPT:-1} ))"

set_or_add() { # key, type, value
  "$PB" -c "Set :$1 $3" "$PLIST" 2>/dev/null || "$PB" -c "Add :$1 $2 $3" "$PLIST"
}

# Marketing + build numbers (Capacitor defaults these to $(MARKETING_VERSION)/
# $(CURRENT_PROJECT_VERSION) build-setting refs; literal overrides are fine).
"$PB" -c "Set :CFBundleShortVersionString $VERSION" "$PLIST"
"$PB" -c "Set :CFBundleVersion $BUILD" "$PLIST"

# Home-screen name.
set_or_add "CFBundleDisplayName" "string" "Heap"

# Portrait-only (mirror Android screenOrientation="portrait").
"$PB" -c "Delete :UISupportedInterfaceOrientations" "$PLIST" 2>/dev/null || true
"$PB" -c "Add :UISupportedInterfaceOrientations array" "$PLIST"
"$PB" -c "Add :UISupportedInterfaceOrientations:0 string UIInterfaceOrientationPortrait" "$PLIST"

# Export compliance: no non-exempt encryption — auto-clears the TestFlight prompt.
set_or_add "ITSAppUsesNonExemptEncryption" "bool" "false"

# AdMob test APP id (iOS). The @capacitor-community/admob pod is installed by
# `cap add ios` regardless of VITE_AD_PROVIDER, and the Google Mobile Ads SDK
# HARD-CRASHES at launch without this key — even though Heap is ad-free here.
set_or_add "GADApplicationIdentifier" "string" "ca-app-pub-3940256099942544~1458002511"

echo "Patched $PLIST: v$VERSION ($BUILD), portrait, encryption=false, ad-free GADApplicationIdentifier"
```

Run: `bash -n scripts/ios-patch-plist.sh`
Expected: no output, exit 0 (syntax valid). PlistBuddy itself is exercised in CI (Task 7).

  desc "Build and upload to TestFlight"
  lane :beta do
    api_key = app_store_connect_api_key(
      key_id: ENV["ASC_KEY_ID"],
      issuer_id: ENV["ASC_ISSUER_ID"],

      scheme: "App",
      export_method: "app-store",
      output_directory: "build/ios",
      clean: true,
      xcargs: "DEVELOPMENT_TEAM=#{ENV['APPLE_TEAM_ID']} -allowProvisioningUpdates",
      export_options: {
        signingStyle: "automatic",
        teamID: ENV["APPLE_TEAM_ID"],
      },
    )

    upload_to_testflight(
      api_key: api_key,
      skip_waiting_for_build_processing: true,
    )
  end
end

## Task 6: GitHub Actions workflow

**Files:**
- Create: `.github/workflows/ios.yml`

macOS job triggered on push to `main` (+ manual dispatch). The guard step gates every later step. Mirrors the Android job in `mobile.yml`. The plist patch runs **after** `cap sync ios` so it has the final word on Info.plist.

- [ ] **Step 1: Write the workflow**

Create `.github/workflows/ios.yml`:

```yaml
name: iOS Build

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  ios:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 2   # need HEAD~1 for the version guard

      - name: Version guard
        id: guard
        run: bash scripts/ios-version-guard.sh

      - uses: actions/setup-node@v4
        if: steps.guard.outputs.should_build == 'true'
        with:
          node-version: 24
          cache: npm

      - name: Install dependencies
        if: steps.guard.outputs.should_build == 'true'
        run: npm ci

      - name: Build web bundle (ad-free)
        if: steps.guard.outputs.should_build == 'true'
        run: npm run build
        env:
          VITE_HEAP_SERVER_URL: ${{ secrets.VITE_HEAP_SERVER_URL }}

      - name: Generate iOS project
        if: steps.guard.outputs.should_build == 'true'
        run: npx cap add ios

      - name: Generate app icons
        if: steps.guard.outputs.should_build == 'true'
        run: npx @capacitor/assets generate --ios

      - name: Sync web assets into iOS
        if: steps.guard.outputs.should_build == 'true'
        run: npx cap sync ios

      - name: Patch Info.plist
        if: steps.guard.outputs.should_build == 'true'
        run: bash scripts/ios-patch-plist.sh

      - name: Build & upload to TestFlight
        if: steps.guard.outputs.should_build == 'true'
        run: fastlane ios beta
        env:
          ASC_KEY_ID: ${{ secrets.ASC_KEY_ID }}
          ASC_ISSUER_ID: ${{ secrets.ASC_ISSUER_ID }}
          ASC_API_KEY_P8: ${{ secrets.ASC_API_KEY_P8 }}
          APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
```
